## Overview

AutoMarket is an online car selling platform that connects private sellers and dealerships with buyers. The backend provides REST APIs for vehicle listing management, user accounts, search and filtering, messaging between parties, payment/escrow coordination, and dealership inventory management. It serves the web frontend and mobile app as a single source of truth.

### Patterns to Use

- **Repository pattern** вЂ” all database access goes through `repositories/`. Services never import SQLAlchemy directly.
- **Service layer** вЂ” all business logic in `services/`. Endpoints are thin wrappers that validate input, call a service, return a response.
- **Dependency injection** вЂ” use FastAPI `Depends()` for DB sessions, current user, permissions. No global state.
- **Async everywhere** вЂ” all endpoints, DB queries (asyncpg), and HTTP calls (httpx) are async. No sync blocking in the event loop.
- **Pydantic v2 models for all I/O** вЂ” strict validation on inbound, explicit serialization on outbound. Use `model_config = ConfigDict(from_attributes=True)` for ORM conversion.
- **Explicit error handling** вЂ” custom exception classes in `exceptions.py`, caught by FastAPI exception handlers. Never return raw 500s.
- **Structured logging** вЂ” JSON logs via `structlog`. Every log line includes `request_id`, `user_id`, `action`.
- **Feature flags** вЂ” use environment variables or a simple DB table for feature toggles. No hardcoded conditionals for rollout logic.

### Patterns to Avoid

- **Fat endpoints** вЂ” if an endpoint function exceeds ~20 lines, the logic belongs in a service.
- **N+1 queries** вЂ” use `selectinload()` / `joinedload()` on relationships. Review all list endpoints for N+1.
- **Raw SQL in services** вЂ” all queries go through repositories. Raw SQL only in migrations or one-off scripts.
- **Mutable default arguments** вЂ” never `def func(items=[])`. Use `None` + internal initialization.
- **Catching bare `Exception`** вЂ” catch specific exceptions. Let unexpected errors propagate to the global handler.
- **Business logic in models** вЂ” SQLAlchemy models are data containers. Validation and logic live in services.
- **Hardcoded secrets** вЂ” all secrets via environment variables or Supabase vault. Never in code, never in config files committed to git.

### Test File Conventions

- Test files mirror source structure: `src/services/vehicle_service.py` в†’ `tests/unit/services/test_vehicle_service.py`
- Prefix all test files with `test_`.
- Use `pytest-asyncio` for all async tests. Mark with `@pytest.mark.asyncio`.
- Use `httpx.AsyncClient` for integration tests against the FastAPI app.
- Use Factory Boy for model factories. Never construct test data inline.
- Integration tests use a real PostgreSQL test database (Docker), not SQLite.
- Each test function tests one behavior. Name format: `test_<action>_<condition>_<expected_result>` (e.g., `test_create_vehicle_missing_vin_returns_422`).
- No test interdependencies. Every test sets up and tears down its own data.

## Security Standards

- All endpoints require authentication except: vehicle search, public listing view, registration, login.
- JWT tokens validated via Supabase Auth. Short-lived access tokens (15 min), long-lived refresh tokens.
- Row-Level Security (RLS) enforced at the PostgreSQL layer via Supabase вЂ” users can only modify their own listings and offers.
- All user input validated by Pydantic before reaching services. No raw user strings in SQL or Elasticsearch queries.
- Image uploads validated: file type whitelist (JPEG, PNG, WebP), max size 10MB, EXIF metadata stripped before storage.
- VIN numbers validated against checksum algorithm before persisting.
- Rate limiting on auth endpoints (5 attempts/min), search (30 req/min), listing creation (10/hour).
- CORS restricted to known frontend origins. No wildcard `*` in production.
- All inter-service communication over HTTPS. Internal services use private network, no public exposure.
- Secrets managed via environment variables. No secrets in code or Docker images.
- SQL injection prevented by parameterized queries (SQLAlchemy ORM). XSS prevented by not rendering user content server-side.
- Dependency scanning via `pip-audit` in CI.
